

# 🎲《骰来骰去》游戏页面 UI 设计方案（Game UI Design Spec）

---

## 1. 设计目标与总体原则

### 1.1 设计目标

* 支持玩家连续完成 **13 回合**而不产生视觉疲劳
* 降低学习成本，让玩家“看界面就知道怎么操作”
* 营造轻桌游氛围，而非工具或表单风格

### 1.2 核心原则

* **信息优先于装饰**
* **阶段状态必须一眼可见**
* **同一时刻只突出一个主要操作**

### 1.3 明确禁止

* 不追求炫酷动画
* 不堆叠强视觉元素
* 不在一个区域承担多个职责

---

## 2. 页面整体结构（三段式）

游戏页面必须采用 **纵向三段式布局**：

```
[ 回合状态区 ]
[ 掷骰核心区 ]
[ 计分选择区 ]
```

三段在视觉上应明显区分（通过留白、卡片、背景层次），即使没有边框也能被一眼识别。

---

## 3. 回合状态区（顶部）

### 3.1 功能定位

该区域是玩家的“导航仪”，用于回答：

* 现在轮到谁？
* 第几回合？
* 当前阶段是什么？
* 还能掷几次骰子？

### 3.2 显示内容（固定）

* 当前玩家（如：玩家 1）
* 当前回合：第 X / 13 回合
* 当前阶段：掷骰阶段 / 计分阶段
* 剩余掷骰次数：X 次

### 3.3 视觉设计要求

* 放入 **白色或半透明圆角卡片**
* 高度克制，不占太多垂直空间
* 文本层级：

  * 玩家 + 回合：最高优先级
  * 阶段 + 剩余次数：次级信息

### 3.4 阶段颜色规范

* 掷骰阶段：蓝色提示
* 计分阶段：绿色提示

---

## 4. 掷骰核心区（视觉与交互核心）

> 页面中最重要区域，必须第一眼吸引注意力

---

### 4.1 骰子展示区

#### 布局要求

* 五枚骰子横向排列
* 居中显示
* 间距一致

#### 骰子视觉规范

* 圆角矩形
* 明确边框
* 轻微阴影（表现立体感）
* 点数建议使用“骰子点阵”（可阶段性实现）

#### 骰子状态区分（必须明显）

**保留状态**必须一眼可识别，允许组合以下方式：

* 上移（像被拿起）
* 边框加粗
* 背景变浅色

> 玩家不应通过“记忆”判断骰子状态，而应通过视觉

---

### 4.2 掷骰操作按钮

#### 核心按钮

* 「摇骰子」
* 「选择计分」

#### 行为规范

* 同一时刻只突出一个主要按钮
* 不可用状态必须明显弱化

#### 颜色规范

* 摇骰子：蓝色主按钮
* 选择计分：绿色主按钮（仅在计分阶段强调）

---

## 5. 计分选择区（信息密集区）

> 信息最多，但视觉必须最稳定的区域

---

### 5.1 整体结构

* 放入一个 **白色大卡片**
* 卡片内部允许滚动
* 顶部可固定标题：**计分表**

---

### 5.2 单条计分项结构（统一）

```
[ 规则名称 ]        [ 预览分数 ]
```

#### 状态区分规范（必须实现）

| 状态   | 视觉表现        |
| ---- | ----------- |
| 已使用  | 灰色文字 + 灰背景  |
| 未使用  | 正常显示        |
| 当前可选 | 高亮边框 / 浅色背景 |

---

### 5.3 数字区 / 组合区分组

建议将计分项分为两组：

* 数字区（1~6 点）
* 组合区（三条 / 四条 / 葫芦 / 顺子 / 快艇 / 全选）

实现方式（任选其一）：

* 细分隔线
* 小标题

---

## 6. 阶段切换的视觉反馈（非常重要）

当阶段从 **掷骰 → 计分** 时，至少发生以下变化之一：

* 状态卡片颜色变化
* 主按钮从“摇骰子”切换为“选择计分”
* 计分区出现高亮提示

> 玩家应“感觉到阶段变了”，而不是读文字才知道

---

## 7. 中景装饰层（统一风格）

### 7.1 装饰类型

* 使用 **透明 PNG / WebP**
* 内容为：计分卡 / 桌游配件剪影
* 无文字、无规则信息

### 7.2 层级关系

```
背景桌布
↓
中景装饰层（弱存在）
↓
UI 卡片与交互元素
```

### 7.3 使用原则

* 强度低于主页面
* 不覆盖任何文字或按钮
* 仅用于“填充氛围”

---

## 8. 节奏与疲劳控制原则

### 8.1 页面节奏

* 每个区域只做一件事
* 每个阶段只有一个主要操作
* 保留足够留白

### 8.2 长时间游玩要求

* 玩到第 10~13 回合仍然清晰
* 不因信息密度产生压迫感

---

## 9. 验收标准（完成定义）

当满足以下条件，视为 UI 实现合格：

1. 新玩家 1 回合内无需询问“下一步做什么”
2. 不看规则也能持续玩完整局
3. 骰子 / 状态 / 计分区域一眼可分
4. 页面像桌游，而不是表单
5. 视觉在多回合下保持稳定、耐看

---

## 10. 给 AI 的实现提示（可直接引用）

> 请严格遵循本 UI 设计文档进行实现
> 不新增未定义的视觉元素
> 不擅自合并区域职责
> 不改变既定信息层级
